  <Accordion title="Why is the number of active devices on WebOS apps lower than the number of opt-ins?">
    During load testing, we found that many LG TVs have a small amount of free RAM and CPU. If several Bright SDK integrated apps are installed on the TV, this may lead to a poor user experience when using the TV. In extreme cases, we encountered UI freezing, for example if the user started playing a 4K video.

    To maintain a high level of user experience, we have limited the number of simultaneously running SDK instances on the TV. Only one SDK will be running, which is the one that was installed first.

    This means if you see a small number of peers with a large number of opt-ins, then most likely most of these TVs already have apps with our SDK installed.
  </Accordion>

  <Accordion title="How does Bright SDK impact my answers to the LG privacy questionnaire?">
    Starting August 2024, LG requires developers to fill in a privacy questionnaire for their apps (same practices used in other app stores e.g. AppStore, Amazon, and Google Play).

    It is important to note that Bright SDK **does not collect** any personal information that can identify or track your users. However, it collects the IP address and performs some crash/performance analytics. Therefore, when you submit an app to the LG Appstore with Bright SDK, in addition to any privacy answers you provide in the questionnaire, please use the following:

    - Privacy Policy → fill in your privacy policy link 
    - Device ID→ Analytics
    - Other Identifier → Analytics
    - Select the country with saved data → U.S.A
    - Shared Data:  Device ID & Other Identifier
